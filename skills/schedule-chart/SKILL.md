# Schedule Chart — 排班与在岗覆盖图

根据用户提供的排班信息（班次名、起止时间、人数），生成：
1. **排班总表**（文字，列出每班的时间、人数、岗位）
2. **全天覆盖图**（HTML 可视化，时间轴 × 班次，色块表示在岗区间）

## 工具名规范

- 读文件 = `file_read`
- 写文件 = `file_write`

## Process

### 1. 收集排班信息

从用户消息中提取以下信息（缺什么问什么，一次问全）：
- **班次名称**：如 早班、午班、晚班、通宵班
- **每班起止时间**：如 7:00-15:00
- **每班人数**：如 3人
- **岗位/角色**（可选）：如 前厅、后厨、打包
- **日期范围**（可选）：如 今天、本周

如果用户提供的是已有排班文件（提到"文件"或给路径），先用 `file_read` 读取。

### 2. 计算覆盖分析

基于排班数据，分析：
- **全天覆盖时间**：最早到岗 → 最晚离岗
- **覆盖缺口**：哪些时段无人或人数不足（低于最小需求）
- **高峰重叠**：哪些时段人最多
- **总工时**：各班次工时汇总

### 3. 生成 HTML 覆盖图

生成一个自包含的 HTML 文件，要求：
- **时间轴**：横轴 0:00-24:00，每小时一格，当前时间用红色竖线标注
- **行**：每人或每班次一行，行首显示班次名+人名（如有）
- **色块**：在岗区间用色块填充，不同班次用不同颜色（早班=蓝、午班=绿、晚班=橙、通宵=紫）
- **底部汇总行**：叠加显示每个时段的在岗总人数
- **缺口高亮**：人数不足的时段用红色背景标注
- **风格**：简洁实用，无外部依赖，纯 HTML+CSS+内联 JS

保存到 `output/排班覆盖图.html`（通过 `file_write` 写入）。

### 4. 输出文字排班表

同时用消息回复用户文字排班表：

```
📋 排班总表

班次     时间         人数  岗位
早班     07:00-15:00   3    前厅2 + 后厨1
午班     11:00-19:00   2    前厅1 + 后厨1
晚班     15:00-23:00   2    前厅1 + 打包1

📊 覆盖分析
- 全天覆盖：07:00-23:00（16小时）
- 高峰时段：11:00-15:00（5人在岗）
- ⚠️ 覆盖缺口：23:00-07:00 无人值守
- 总工时：20人·时
```

## Important Principles

- **先问再做**：信息不全时一次性问全，不要来回追问
- **可视化优先**：HTML 覆盖图是核心交付物，必须生成并写入文件
- **实用不花哨**：图表追求信息密度和可读性，不做装饰性设计
- **时区明确**：所有时间默认用户当地时区，不做时区转换

# 包车报价

当用户表达包车/租车/团建用车/接送等意图时，按以下流程处理。

## 第一步：收集信息

必须确认以下信息才能报价（缺少的逐一询问，不要一次问太多）：

- 出发地、目的地（用于判断距离档）
- 用车日期
- 人数（用于推荐车型）
- 联系电话

可选信息（不主动问，用户提到时记录）：
- 单程/往返（默认往返）
- 车型偏好（不指定则按人数推荐）
- 是否需要发票/住宿

### 人数与车型推荐

| 人数 | 推荐座位 | 说明 |
|------|----------|------|
| 1~4人 | 5座 | 轿车/SUV |
| 5~6人 | 7座 | 商务车 |
| 7~9人 | 10座 | 小型车 |
| 10~15人 | 16座或17座 | 中巴 |
| 16~21人 | 22座 | 中巴 |
| 22~26人 | 27座 | 中巴 |
| 27~33人 | 34座 | 大巴 |
| 34~37人 | 38座 | 大巴 |
| 38~45人 | 46座 | 大巴 |
| 46人以上 | 53座 | 大巴，或多车 |

注意：只推荐上述有定价规则的座位数（5/7/10/16/17/22/27/34/38/46/53），不要编造不存在的车型。

## 第二步：报价

### ⚠️ 必须先调用高德地图

**在报价之前，必须先调用 `amap_driving` 获取精确里程和过路费。不调用此工具直接报价是错误的。**

```
amap_driving(origin="出发地", destination="目的地")
```

返回数据包括：
- `distance_km`：实际里程（公里）
- `tolls`：过路费（元）
- `distance_tier`：自动判断的距离档（市内/近郊/远郊/长途）

## 第三步：谈价与确认

报价后进入协商阶段，用户可能：

- **还价/嫌贵** → 说明"参考价是系统定价，多天或长期合作可联系负责人谈优惠 18011977623"
- **问更多细节**（车型/司机/发票/住宿等）→ 正常回答
- **修改需求**（换人数/日期/路线）→ 重新报价
- **补充信息**（加行李、改时间等）→ 记录并更新报价

### 确认意向

当用户没有新问题后，主动询问确认：

「这个方案您看可以吗？如果没有其他要补充的，我帮您生成确认链接～」

用户回复"可以"/"确认"/"订吧"/"好的就这样"等 → 视为确认意向

### 发确认链接

确认意向后，回复用户确认页面链接（信息已预填好，用户只需选择全款或定金并确认）：

```
您的包车订单信息已确认，请点击链接完成下单：
https://www.86bus.com/charter_confirm?order_id=xxx

选择支付方式：全款 或 定金
```

> 注意：确认链接页面需要后台开发，目前暂用文字引导，等接口就绪后替换为真实链接。

## 第四步：推送通知与找车队

### Push 1：收到联系方式后

拿到用户联系电话后，输出以下标记（内容会推送到包车负责人个人微信）：

```
[NOTIFY:charter_lead]客户姓名/称呼，电话，用车日期，出发地→目的地，人数，推荐车型，往返/单程，报价￥xxx起[/NOTIFY]
```

### Push 2：用户完成确认页下单后

用户在确认页面选择全款/定金并确认后，输出：

```
[NOTIFY:charter_confirmed]订单确认：客户姓名，电话，日期，路线，车型，行程，报价，支付方式（全款/定金）[/NOTIFY]
```

### Push 3：找到车队后

确认后立即调用 car-finder-v2 找车队：

```
a2a_send(agent="car-finder-v2", message="需要XX座车型，X月X日，出发地→目的地，单程/往返")
```

收到车队信息后，回复用户车队概况，并输出推送标记：

```
[NOTIFY:charter_fleet]已匹配车队：车队名称，可用车型数量，联系电话[/NOTIFY]
```

### 超时介入：用户一直未确认

如果报价后用户迟迟未确认（超过合理时间），通知包车负责人介入：

```
[NOTIFY:charter_timeout]包车咨询超时未确认：客户姓名，电话，日期，路线，报价。请负责人跟进。[/NOTIFY]
```

同时回复用户：「您好，如果您还需要包车服务，可以随时联系我们的包车负责人：18011977623（微信同号），他会为您安排～」

## 报价示例

用户："我想包个车，30人，从广州去佛山，下周六"

回复：

```
好的，30人推荐34座或38座大巴。广州到佛山属于近郊，给您报个参考价：

🚗 包车报价参考：
路线：广州→佛山（近郊）
车型：34座大巴
行程：往返
参考价：￥880起

⚠️ 以上仅供参考，最终价格以实际确认为准。

请问您是单程还是往返？方便留个联系电话吗？
```

用户回复电话和行程后：

```
收到！我整理一下您的信息：
- 出发地→目的地：广州→佛山
- 日期：下周六
- 人数：30人
- 车型：34座大巴
- 行程：往返
- 参考价：￥880起

这个方案您接受吗？还有什么要补充的？如果没有问题，我帮您生成确认链接～
```

用户确认后：

```
您的包车订单信息已确认，请点击链接完成下单：
https://www.86bus.com/charter_confirm?order_id=xxx

选择支付方式：全款 或 定金
```

## 注意事项

- **所有报价必须标注"仅供参考，以实际确认为准"**
- **必须调用 `amap_driving` 获取精确里程和过路费**，不调用直接报价是严重错误
- 报价必须包含里程和过路费数据，例如："全程约147公里，高速费67元"
- 不随意估价，严格按价目表报价
- 精确报价公式：每日费用 = max(里程×公里单价 + 过路费×路桥系数 + 基础费 + 司机费, 底价)
- 不议价，但可说明"多天/长期合作可联系负责人谈优惠"
- **报价后必须主动询问确认**：不要只报完价就结束，要问"这个方案您接受吗？"
- 包车负责人电话：18011977623（微信同号），如用户要求直接联系可提供
- 微信客服消息有48小时窗口限制，超时无法主动发送
